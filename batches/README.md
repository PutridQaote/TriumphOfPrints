# Batch file roles

- `canonical-airdrop.yaml` is the source of truth for the 333 ordered print
  definitions. Destinations are intentionally null. Its parent is the existing
  mainnet inscription and its postage is the approved 1,000 sats.
- `mainnet/js.yaml` inscribes the renderer. Mainnet recipient batches are
  generated by `just refresh-airdrop`; do not add destinations by hand.
- `regtest/` contains only regtest inputs. The parent JPEG is a synthetic
  stand-in so parent/child provenance can be rehearsed without mainnet assets.
- `archive/2026-02-18/` preserves the February holder snapshot batch and its
  print-to-Ephemera mapping. Its destinations are historical and must not be
  used for inscription.
- `generated/` is ignored. Every refresh creates a timestamped candidate bundle
  containing an airdrop YAML, updated mapping CSV, and validation summary.

The recursive child payload remains at the repository root as `inscribed.html`.
It is a mainnet release artifact and will be updated only after `main.js` has a
confirmed mainnet inscription ID. Regtest uses a generated copy under `env/`.
